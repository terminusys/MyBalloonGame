# 🚀 发布你的热气球游戏到公网

## 🎯 方法 1：Netlify Drop（最简单！⭐推荐）

### 步骤：

1. **访问 Netlify Drop**
   ```
   https://app.netlify.com/drop
   ```

2. **拖拽文件**
   - 打开文件夹 `F:\MyGame`
   - 直接将 `index.html` 拖到浏览器的 Netlify Drop 页面中
   - 或者拖拽整个 `MyGame` 文件夹

3. **获取链接**
   - 上传完成后会自动生成一个链接
   - 类似：`https://random-name-123456.netlify.app`
   - 这个链接可以直接分享给朋友！

4. **优点**
   - ✅ 不需要注册账号
   - ✅ 完全免费
   - ✅ 支持 HTTPS
   - ✅ 全球 CDN 加速
   - ✅ 立即生效

---

## 🎯 方法 2：GitHub Pages（永久免费）

### 步骤：

1. **创建 GitHub 仓库**
   ```bash
   # 在终端执行
   cd F:\MyGame
   git init
   git add index.html
   git commit -m "Initial commit: 热气球游戏"
   ```

2. **推送到 GitHub**
   - 登录 https://github.com
   - 创建新仓库（Repository）
   - 按照提示推送代码

3. **启用 GitHub Pages**
   - 进入仓库的 Settings
   - 找到 Pages 选项
   - Source 选择 `main` 分支
   - 保存

4. **访问地址**
   ```
   https://你的用户名.github.io/仓库名/
   ```

## 🎯 方法 3：Vercel（推荐给开发者）

### 步骤：

1. **安装 Vercel CLI**
   ```bash
   npm install -g vercel
   ```

2. **部署**
   ```bash
   cd F:\MyGame
   vercel
   ```

3. **按照提示操作**
   - 登录 Vercel（会自动打开浏览器）
   - 回答几个问题
   - 等待部署完成

4. **获取链接**
   - 部署完成后会显示链接
   - 类似：`https://your-project.vercel.app`

---

## 🎯 方法 4：Surge.sh（超快！）

### 步骤：

1. **安装 Surge**
   ```bash
   npm install -g surge
   ```

2. **部署**
   ```bash
   cd F:\MyGame
   surge
   ```

3. **设置域名**
   - 输入邮箱（首次使用）
   - 设置密码
   - 输入项目路径：直接回车（当前目录）
   - 输入域名：`my-balloon-game.surge.sh`（或其他名字）

4. **访问地址**
   ```
   https://my-balloon-game.surge.sh
   ```

## ⚠️ 重要提示

### Supabase 多人同步功能：

如果你想让多人实时同步功能正常工作，需要：

1. ✅ 确保 `index.html` 中的 Supabase key 是正确的
2. ✅ 确保在 Supabase 中运行了 `setup_supabase.sql` 脚本
3. ✅ 确保 Supabase 的 Realtime 功能已启用

检查步骤：
- Supabase Dashboard → Database → Replication
- 确保 `balloon_positions` 表的 Realtime 是开启状态

## 🎮 测试步骤

1. 在浏览器打开你的公网链接
2. 按 F12 打开控制台
3. 应该看到：
   ```
   🎈 当前玩家: 飞行员 XXX
   ✅ Supabase 客户端初始化成功
   🔌 初始化 Supabase Realtime 连接...
   ✅ Realtime 连接成功！
   ```

4. 在另一个浏览器标签页或手机上打开同样的链接
5. 点击移动气球，应该能在两个设备上看到实时同步

## 🆘 如果遇到问题

### 问题 1：其他人访问显示空白
- 检查浏览器控制台的错误信息
- 确保 Supabase URL 和 Key 正确

### 问题 2：多人同步不工作
- 检查 Supabase Realtime 是否启用
- 检查 `balloon_positions` 表是否创建成功
- 检查 RLS 策略是否正确设置

### 问题 3：加载很慢
- 这是正常的，因为需要加载 Phaser.js 和 Supabase.js 库
- 首次加载可能需要几秒钟
